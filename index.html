RFC-3629 UTF-8 Validation Requirements
NON-NEGOTIABLE CONSTRAINTS (Red Channel)
  • Reject overlong sequences (security critical)
  • Forbid surrogate pairs U+D800–U+DFFF
  • Forbid codepoints > U+10FFFF
  • Reject lone continuation bytes
  • Reject truncated sequences
  • Reject noncharacters in all planes
Critical Requirement: On any invalid subsequence, emit exactly one U+FFFD and record precise byte indices [start, end) in global coordinates.
STREAMING NFC NORMALIZATION (Blue Channel)
  • Do NOT normalize per chunk
  • Maintain buffer of current canonical segment
  • Flush only on seeing a starter (combining class 0)
  • Handle up to 64 consecutive non-starters
VERIFICATION REQUIREMENTS (Interpretant Layer)
  • No false accepts of overlongs
  • Correct cross-chunk composition (A + U+0301)
  • Exact span for invalid sequences
  • No double emission of U+FFFD

The Missing Dimension

Current AI operates with dyadic reasoning: Signal → Response.

This creates the "jailing" problem where plausible patterns override correct answers (JAILING vs JINGALL, accepting overlongs, corrupting boundaries).

Nepsis adds triadic reasoning: Signal → Object → Interpretant

The interpretant verifies that the response actually satisfies the constraints, not just that it matches a likely pattern.
